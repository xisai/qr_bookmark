# アプリの概要と特徴

## プラットフォーム
Flutter製で、PWA対応のWebアプリ。

## 対応言語
英語・日本語のマルチランゲージ対応。ブラウザの言語設定に基づいて表示言語を切り替える。主な利用者は日本語ユーザーを想定。

## QRコード生成
テキストやバイナリデータからQRコードを作成する。
テキストの場合、文字エンコードはUTF-8である。
バイナリの場合、"BEEFFEEB01"といった16進数文字列からバイナリに変換する。16進数以外の文字や奇数桁などの不正入力はエラーとする。

## データ保持
QRコード生成にかかわるデータはURLのパスセグメントに格納する。QR生成をしたURLをブックマークし、そのブックマークを開くとQRが生成された状態で表示される。

URLの形式：`https://example.com/qr/<encoded>[/<sizeSteps>]`

パスセグメントのエンコード手順（あいことばなし）：
1. ランダム8文字のソルトを生成する
2. データをバイト列に変換し、ソルトのバイト列と繰り返しXOR演算する
3. XOR後のバイト列をBase64URLエンコードする
4. ソルトをエンコード結果の先頭に付与してパスセグメントとする

復元時はソルトをパスセグメント先頭から取り出し、逆順に処理する。

## あいことば（パスフレーズ保護）
QRコード生成時にあいことばを設定すると、URLだけではQRデータを復元できなくなる。
あいことばを知っている人だけがQRコードを表示できる。

あいことばありのエンコード手順：
1. データバイト列に対して HMAC-SHA256(key=あいことば) の先頭8バイトをMACとして計算する
2. MAC + データ バイト列を あいことば で繰り返しXOR演算する
3. 先頭にマーカーバイト（0xFF）を付与する
4. 以降は通常のソルトXOR + Base64URLエンコード手順を適用する

復元時にマーカーバイトを検出した場合はあいことばを要求し、HMACを検証する。

あいことばの要件：6文字以上。

## PWAとしての挙動
iOS・Androidで「ホーム画面に追加」をするとホーム画面にアイコンを追加できる。
- QR生成後にホーム画面に追加した場合：そのQRコード画像がアイコンになる
- QR生成前（QR生成画面）でホーム画面に追加した場合：デフォルトのアプリアイコンになる

iOSの制約上、ホーム画面に追加した時点のアイコンが固定されるため、後からQRを変更してもアイコンは変わらない。

### iOSホーム画面URLの保持
iOSで「ホーム画面に追加」した際にQRデータを含むURLを保持するため、以下の対策を実装している。

- **パスベースURL**: ハッシュフラグメント（`#`以降）はiOSに除去されるため、`usePathUrlStrategy()` でパスベースルーティングを採用
- **Service Worker**: `sw.js` が `manifest.json` のフェッチをインターセプトし、`start_url` を現在のQR URLに書き換える
- **GitHub Pages対応**: 存在しないパスへのアクセスは `404.html` で処理し、アプリのルートにリダイレクト後にパスを復元する

### Androidホーム画面アイコン
Android Chrome は `manifest.json` の `icons` フィールドを使うが、data URL を直接 icons に指定すると Chrome のPWAインストール判定を通過できない。そのため以下の方式で実装している。

- **SW キャッシュ＋仮想 URL**: QR PNG を SW の CacheStorage に格納し、`/icons/qr-dynamic-icon.png` という仮想 URL として SW から配信する
- **manifest 書き換え**: SW が `manifest.json` をインターセプトする際、QRアイコンがキャッシュ済みであれば `icons` を `/icons/qr-dynamic-icon.png` に書き換える
- **blob manifest 連動**: `index.html` の `_qrIconMode` フラグが `true` のとき、blob manifest の `icons` も同じ仮想 URL に切り替わる
- **フォールバック**: CacheStorage にQRアイコンがない場合は `Icon-512.png` を返す

# 画面

## 共通
全画面にハンバーガーメニューを設置し、各画面へのナビゲーションを提供する。
ハンバーガーメニューの最下部にバージョン番号を表示する。

## QR表示画面
URLパスに `/qr/<encoded>` が含まれる場合にこのページが表示される。

表示状態は3種類：
- **locked**：あいことば保護されたURLを直接開いた場合。あいことば入力UIを表示し、正しいあいことばを入力するとQRを表示する。
- **unlocked**：あいことばを手動入力して復号に成功した状態。QRを表示する。
- **autoUnlocked**：QR生成画面から遷移直後（あいことばが自動引き渡された）の状態。QRを表示し、あいことば設定済みを示すバナーを表示する。

以下の要素が存在する（unlocked / autoUnlocked 時）：
- QR画像（現在のサイズで生成・表示）
- QR画像拡大ボタン・縮小ボタン
  - ボタンを押すたびに10%単位でサイズを変更し、QR画像を再生成する
  - 最大サイズ：マージンを考慮した画面幅いっぱい
  - 変化量（10%）は `AppConstants.qrSizeStepFactor` で調整可能
  - 現在のサイズステップはURLパスの第2セグメント（`/qr/<encoded>/<sizeSteps>`）として保持される
  - サイズ変更後もあいことばは保持され、locked状態に戻ることはない

## QR生成画面
`/` に遷移するとこのページが表示される。
以下のフォームが存在する：
- テキスト・バイナリ選択コンボボックス
- QRデータ入力テキストボックス
- あいことば入力テキストボックス（任意・6文字以上）
- QR生成ボタン

QR生成ボタンを押下すると、入力値のバリデーション後QRを生成し、`/qr/<encoded>` のURLに遷移する（QR表示画面へ）。あいことばを入力した場合は遷移先でそのままQRが表示される（autoUnlocked状態）。
